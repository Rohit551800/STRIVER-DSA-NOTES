1️⃣ Brute Force → Optimization

📌 Core Idea:

Simplest solution first → then optimize using:
• HashMap (O(1) lookup)
• Prefix Sum (avoid repeated calculations)
• Sliding Window

🧠 Thinking Steps:

1 Brute force soch
2 Time complexity nikalo (O(n²), O(n³))
3 Repeated work identify karo
4 Optimize: hashmap/prefix sum use karo

💡 Example: Subarray Sum = K

Brute: O(n²) → check every subarray
Optimized: O(n) → prefix sum + hashmap
prefixSum[j] - prefixSum[i] = K
→ find prefixSum[i] = prefixSum[j] - K in map

2️⃣ Two Pointer Pattern

📌 Core Idea:

2 pointers: left (start) + right (end)
🧠 Kab use kare?
✓ Sorted array
✓ Pair/triplet problems
✓ Palindrome

🧠 Thinking Steps:

1 Array sorted hai? → two pointer use karo
2 Pair find karna hai?
3 Move logic:
   • sum > target → right--
   • sum < target → left++

💡 Example: Two Sum (Sorted)

left = 0, right = n-1
while(left < right):
  sum = arr[left] + arr[right]
  if sum == target → return
  elif sum < target → left++
  else → right--

3️⃣ Sliding Window

🧠 Thinking Steps:

Case 1: Fixed Window (size = k)

Case 2: Variable Window
1 right expand

3 update answer

💡 Example: Longest Substring

4️⃣ Prefix Sum

📌 Core Idea:

Repeated sum calculation avoid karna

🧠 Thinking Steps:

1 Prefix array banao:
   prefix[i] = sum(0 → i)
2 Range sum:
   sum(l → r) = prefix[r] - prefix[l-1]

💡 Example: Subarray Sum = K

prefix[j] - prefix[i] = k
→ prefix[i] = prefix[j] - k
→ use hashmap for O(1) lookup

7️⃣ Binary Search

📌 Core Idea:

Search space ko half karna

🧠 Thinking Steps:

1 Array sorted hai?
2 Answer space define karo
3 mid calculate
4 Condition check:
   • left ya right move karo

💡 Template:

left = 0, right = n-1
while(left <= right):
  mid = left + (right - left) // 2
  if arr[mid] == target → return mid
  elif arr[mid] < target → left = mid + 1
  else → right = mid - 1
